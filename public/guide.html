☝️ One-Click Deploy (Sabse Easy)

Requirement: Sirf GitHub account aur free Cloudflare account. Credit card ki zarurat nahi.

1

Deploy Button Click Karo

Neeche button par click karo. Ye tumhare GitHub aur Cloudflare account mein copy banayega.

Deploy to Cloudflare
2

Cloudflare mein Login Karo

Apna Cloudflare account use karo. Ye tumhare account mein deploy hoga, kisi aur ke nahi.

3

Project Name Choose Karo

Example: formforge. D1 database name automatically formforge-db banegi.

4

AUTH_SECRET Daalo

Deploy form mein AUTH_SECRET field dikhega. Value generate karo:

openssl rand -hex 32

Output copy-paste karo. Agar blank chhod diya to baad mein yahan se fix karna hoga.

5

RESEND fields (Optional)

Email chahiye to daalo, nahi to blank chhodo. Email notifications automatically disable ho jayenge.

6

Deploy Karo

Cloudflare D1 database auto-create karega. FormForge tables first request par auto-create karega. 2-3 minute mein live!

7

Dashboard Open Karo

URL milega jaise https://formforge-YOUR.workers.dev. Uske baad /dashboard lagao aur account banao.

🔑 AUTH_SECRET Kya Hai & Kaise Set Karein?

AUTH_SECRET ek secret key hai jo sessions aur API keys ko secure karne ke liye use hoti hai. Ye required hai. Bina iske account create nahi hoga.

Agar deploy ke time blank chhod diya → ab fix karo:

1

Terminal mein run karo: openssl rand -hex 32

2

Cloudflare Dashboard → Workers & Pages → apna Worker click karo.

3

Settings → Variables & Secrets → Add variable → Secret select karo.

4

Name: AUTH_SECRET, Value: terminal se copy kiya hua secret. Save karo.

5

Redeploy karo Worker ko, phir dashboard reload karo. Ab register karo.

🛠 Manual Deploy

git clone https://github.com/adrianak2026/FormForge.git
cd FormForge
npm install
npx wrangler login
npx wrangler secret put AUTH_SECRET
npm run deploy

D1 auto-provision hota hai. Migrations optional hain kyunki app first request par schema create kar leta hai.

🌍 Environment Variables

NameRequired?What to enter
AUTH_SECRETYesopenssl rand -hex 32 output. Cloudflare deploy form mein dikhega.
RESEND_API_KEYNoresend.com se free API key. Blank = email off.
RESEND_FROMNoSender address. Example: FormForge <forms@example.com>
DBYesD1 binding name — already set in wrangler.jsonc. Secret nahi hai.

DATABASE_URL mat dalo. FormForge PostgreSQL use nahi karta.

📊 Dashboard Kaise Use Karein

  • Forms tab: + New se form banao, search karo, select karo
  • Submissions: Expand karo, JSON view, status (accepted/spam), time-ago
  • Settings tab: Form name, origins, honeypot, webhook, email configure karo
  • CSV Export: ⬇ button se saari submissions download karo
  • API Keys: Read-only tokens banao programmatic access ke liye

🔒 Privacy & Security

  • Data sirf tumhare Cloudflare D1 mein rehta hai
  • IP addresses SHA-256 hash karke store hote hain
  • Passwords PBKDF2-SHA256 (210K iterations) se hashed
  • Session tokens HMAC-SHA256 hash karke store
  • No third-party analytics/tracking
  • Email/webhook optional, off by default

🐛 Troubleshooting

"Set AUTH_SECRET before creating accounts"

Solution: AUTH_SECRET set karo Cloudflare secrets mein aur redeploy karo.

"D1_NOT_CONFIGURED"

Cloudflare Dashboard mein Worker → Settings → Bindings → D1 database formforge-db bind karo.

Guide page download ho raha tha

Ab /guide.html web page hai. Markdown version /guide.html bhi available hai.
